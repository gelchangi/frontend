‚ùì Potential Questions & Answers

Q: Why Vue 2 instead of Vue 3?

A: Course requirement specified Vue 2 only.

Q: Why not use Vue Router?

A: Requirements state to use v-if for page switching. This keeps it simple and meets the spec.

Q: How does search work?

A: On every keystroke, it calls the backend with GET /lessons?q=. Backend filters and returns results.

Q: How is state managed?

A: All state is in the Vue data object: lessons, cart, currentPage, user info, etc. No Vuex needed for this scope.

Q: Is the cart persisted?

A: No, cart is in-memory only. Requirements don't ask for persistence. Only theme preference is saved to localStorage.

Q: How are spaces updated?

A: After POST /orders succeeds, the app loops through cart items and makes PUT /lessons/:id requests to decrement spaces.
